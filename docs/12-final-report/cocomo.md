# Оценка трудозатрат — COCOMO I (Basic)

## Проект: MindFlow

### 1. Исходные данные

**Тип проекта:** Embedded (встроенный) — мобильное приложение с нетривиальными требованиями к безопасности, офлайн-режиму и архитектурным ограничениям PCMEF.

**Измеренный объём кода (фактический):**

| Компонент | Язык | Строки кода (SLOC) |
|-----------|------|-------------------|
| Backend (Java) — Entity, Foundation, Mediator, Control, Security | Java | ~2 400 |
| Android — Foundation, Mediator, Control, Presentation, Navigation | Kotlin | ~4 800 |
| Тесты (backend JUnit + Android JUnit) | Java/Kotlin | ~1 600 |
| SQL (DDL + data.sql) | SQL | ~200 |
| **Итого** | | **~9 000 SLOC** |

Для COCOMO используем **KDSI = 9.0** (тысяч строк исходного кода).

---

### 2. Базовые коэффициенты COCOMO (Embedded mode)

| Параметр | Значение |
|----------|---------|
| Режим | Embedded |
| a | 3.6 |
| b | 1.20 |
| c | 2.50 |
| d | 0.32 |

---

### 3. Расчёт трудозатрат

**Формула:** `E = a × (KDSI)^b`

```
E = 3.6 × (9.0)^1.20
  = 3.6 × 13.97
  = 50.3 человеко-месяца
```

---

### 4. Расчёт длительности

**Формула:** `T = c × E^d`

```
T = 2.50 × (50.3)^0.32
  = 2.50 × 3.83
  = 9.6 месяцев (для полной команды)
```

Для одного разработчика с неполной занятостью:

```
Фактический срок = 4.5 мес. (18 нед.)
Средняя нагрузка = 50.3 / 4.5 ≈ 11.2 ч/нед. ≈ 1.4 ч/день
```

---

### 5. Оценка стоимости (условная)

| Параметр | Значение |
|----------|---------|
| Трудозатраты | ~156 реальных часов |
| Ставка junior-разработчика (РФ, 2026) | ~600 руб./час |
| **Условная стоимость разработки** | **~93 600 руб. ≈ 93.6 тыс. руб.** |

---

### 6. Распределение трудозатрат по этапам (по COCOMO)

| Фаза | % от общих | Часы |
|------|-----------|------|
| Анализ требований | 8% | 12.5 |
| Проектирование | 18% | 28.1 |
| Реализация | 40% | 62.4 |
| Тестирование | 22% | 34.3 |
| Интеграция и документация | 12% | 18.7 |
| **ИТОГО** | **100%** | **156 ч** |

---

### 7. Факторы корректировки (EAF — Effort Adjustment Factor)

| Фактор | Оценка | Коэффициент |
|--------|--------|------------|
| Надёжность ПО (RELY) | Высокая (медицина/здоровье) | 1.15 |
| Размер БД (DATA) | Низкий (<300 МБ) | 0.94 |
| Сложность продукта (CPLX) | Высокая (mobile + JWT + offline) | 1.30 |
| Опыт разработчика (VEXP) | Низкий (студент) | 1.29 |
| Инструменты (TOOL) | Современные (Android Studio, IntelliJ) | 0.83 |
| **EAF = произведение** | | **≈ 1.51** |

**Скорректированные трудозатраты:** `E_adj = 50.3 × 1.51 ≈ 76 ч/м`

---

### 8. Выводы

- Проект относится к категории **средней сложности** по COCOMO Embedded.
- Фактические 156 часов соответствуют скорректированной оценке для одного студента-разработчика.
- Основной вклад в трудозатраты — **реализация (40%)** и **тестирование (22%)**, что соответствует реальному распределению.
