## 1. Введение

### 1.1 Наименование системы
**Мобильное приложение для поддержки ментального здоровья «MindFlow»** — клиент-серверная система, включающая Android-приложение и REST API-сервер.

### 1.2 Краткая характеристика области применения
Система предназначена для поддержки психоэмоционального состояния пользователей путём предоставления медитаций, дневника настроения и аналитики. Относится к категории wellness-приложений (здоровье и благополучие).

### 1.3 Основание для разработки
Курсовой проект по дисциплине «Программная инженерия», направление 09.03.04, СКФУ, 2026 г.

---

## 2. Назначение и цели создания системы

### 2.1 Назначение
Система предназначена для:
- индивидуального мониторинга психоэмоционального состояния;
- доступа к библиотеке медитаций, структурированной по категориям;
- формирования долгосрочной привычки осознанности через ежедневные практики.

### 2.2 Цели создания
1. Снизить порог входа в практику медитации для начинающих.
2. Предоставить инструмент для регулярного отслеживания настроения.
3. Обеспечить доступ к контенту в офлайн-режиме.
4. Гарантировать безопасность персональных данных пользователей.

### 3.2 Требования к архитектуре

#### 3.2.1 Серверная часть
Должна реализовывать паттерн **PCMEF**:
- **Control** — REST-контроллеры, обработка HTTP-запросов, валидация входных данных
- **Mediator** — бизнес-логика, управление транзакциями
- **Entity** — JPA-сущности с бизнес-методами
- **Foundation** — репозитории Spring Data JPA

#### 3.2.2 Клиентская часть (Android)
Реализует адаптацию PCMEF:
- **Presentation** — Jetpack Compose экраны
- **Control** — ViewModel + StateFlow
- **Mediator** — Repository-интерфейсы
- **Foundation** — Room DAO + Retrofit API

#### 3.2.3 Правило зависимостей
Зависимости направлены строго сверху вниз: P → C → M → E/F. Обратные зависимости запрещены.

### 3.4 Требования к безопасности

| Код | Требование |
|-----|------------|
| S-01 | Пароли хранятся в виде BCrypt-хешей (strength = 12) |
| S-02 | Аутентификация через JWT: access-token (15 мин), refresh-token (7 дней) |
| S-03 | Все защищённые эндпоинты требуют заголовок `Authorization: Bearer <token>` |
| S-04 | JWT-токены хранятся на клиенте в шифрованном DataStore |
| S-05 | Реализована ролевая модель: ROLE_USER, ROLE_ADMIN |
| S-06 | CORS настроен для разрешённых источников |

## 8. Критерии приёмки

Система считается принятой при выполнении всех следующих условий:
1. Все эндпоинты из раздела 3.3 функционируют корректно
2. Android-приложение запускается на устройстве с Android 8.0+
3. Покрытие тестами ≥ 40% по строкам кода
4. Документация OpenAPI доступна по `/swagger-ui.html`
5. Данные сохраняются в офлайн-режиме через Room
6. JWT-аутентификация работает (login → token → protected request)
