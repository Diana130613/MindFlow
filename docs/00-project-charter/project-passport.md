## 2. БИЗНЕС-КОНТЕКСТ

### Проблема
В современном мире уровень стресса, тревожности и эмоционального выгорания
неуклонно растёт: по данным ВОЗ, каждый четвёртый человек сталкивается
с расстройствами психического здоровья. Существующие решения либо
слишком дороги, либо требуют постоянного подключения к интернету,
либо не адаптированы к индивидуальным потребностям пользователя.

### Решение
MindFlow - мобильное приложение, которое предоставляет персонализированные
медитации, техники дыхания, дневник настроения и аналитику эмоционального
состояния. Приложение работает в офлайн-режиме и использует JWT-аутентификацию
для синхронизации данных с облачным сервером. Архитектура построена на паттерне
PCMEF с разделением клиентской части (Android) и серверной (Spring Boot + PostgreSQL).

## 3. ЦЕЛИ ПРОЕКТА

- **Цель 1:** Разработать Android-приложение с минимум 5 экранами для практики
  медитации и отслеживания ментального здоровья
- **Цель 2:** Создать REST API на Spring Boot с JWT-аутентификацией (8+ эндпоинтов)
- **Цель 3:** Обеспечить офлайн-режим через локальное кэширование (Room DB)
- **Цель 4:** Реализовать архитектуру PCMEF с чистым разделением слоёв

## 5. КЛЮЧЕВЫЕ РИСКИ

| Риск | Вероятность | Стратегия реагирования |
|------|------------|----------------------|
| Сложность интеграции JWT на Android | Средняя | Использование готовой библиотеки OkHttp interceptor |
| Проблемы с настройкой PostgreSQL | Низкая | Использование Docker для локальной БД |
| Нехватка времени на тестирование | Средняя | Параллельное написание тестов во время разработки |
| Сложность реализации офлайн-синхронизации | Высокая | Использование Room + WorkManager |

## 6. ТЕХНОЛОГИЧЕСКИЙ СТЕК

### Клиентская часть (Android)
- Язык: Kotlin
- UI: Jetpack Compose + Material Design 3
- Архитектура: MVVM (адаптация PCMEF)
- Сеть: Retrofit 2 + OkHttp
- Кэш: Room Database
- DI: Hilt
- Состояние: StateFlow + ViewModel

### Серверная часть (Backend)
- Язык: Java 17
- Фреймворк: Spring Boot 3.x
- БД: PostgreSQL 15
- ORM: Spring Data JPA + Hibernate
- Безопасность: Spring Security + JWT
- Документация: Swagger/OpenAPI 3.0
- Сборка: Maven
