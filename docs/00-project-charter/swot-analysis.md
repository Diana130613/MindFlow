# SWOT-АНАЛИЗ

## Текущее состояние рынка приложений для ментального здоровья

### Сильные стороны (Strengths)
| # | Фактор |
|---|--------|
| 1 | Высокий спрос на инструменты управления стрессом |
| 2 | Доступность Android-платформы (70%+ рынка) |
| 3 | Зрелые технологии: Kotlin, Spring Boot, PostgreSQL |
| 4 | Возможность работы офлайн (Room Database) |
| 5 | Современная архитектура PCMEF обеспечивает масштабируемость |

### Слабые стороны (Weaknesses)
| # | Фактор |
|---|--------|
| 1 | Ограниченный бюджет (учебный проект без реального финансирования) |
| 2 | Один разработчик — риск перегрузки |
| 3 | Отсутствие реальных аудиозаписей медитаций (используем текстовые сценарии) |
| 4 | Нет возможности клинической верификации методик |

### Возможности (Opportunities)
| # | Фактор |
|---|--------|
| 1 | Растущий рынок wellness-приложений (+25% ежегодно) |
| 2 | Интеграция с wearables (умные часы) в будущем |
| 3 | Добавление AI-персонализации рекомендаций |
| 4 | Публикация в Google Play как реальный продукт |

### Угрозы (Threats)
| # | Фактор |
|---|--------|
| 1 | Конкуренция с Headspace, Calm, Insight Timer |
| 2 | Риск нарушения GDPR при хранении данных о здоровье |
| 3 | Зависимость от сторонних библиотек (устаревание) |
| 4 | Ограниченное время разработки (18 недель) |

### Стратегические выводы
- **SO:** Использовать зрелый стек для быстрой разработки MVP с фокусом на офлайн-функциональность
- **ST:** Дифференцироваться за счёт офлайн-режима и приватности данных
- **WO:** Начать с текстовых медитаций, добавить аудио в v2.0
- **WT:** Строго следовать PCMEF для поддерживаемости кода
