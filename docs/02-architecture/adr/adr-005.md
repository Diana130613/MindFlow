# ADR-005: UI-фреймворк Android — Jetpack Compose
 
Статус: ✅ Принято

---

## Контекст

Необходимо выбрать подход к разработке UI для Android-приложения.
Требования:
- Современный инструмент с долгосрочной поддержкой
- Совместимость с ViewModel и StateFlow
- Поддержка Material Design 3
- Декларативный подход для упрощения кода

---

## Решение

Использовать **Jetpack Compose** — декларативный UI-фреймворк от Google.

---

## Обоснование

| Критерий          | Jetpack Compose                                  |
|-------------------|--------------------------------------------------|
| Поддержка Google  | Рекомендован как стандарт с 2021 года            |
| Декларативность   | UI = f(state) — проще рассуждать о состоянии     |
| Объём кода        | Значительно меньше, чем XML + View               |
| Material Design 3 | Нативная поддержка актуального дизайна           |
| Интеграция        | Нативная поддержка ViewModel, StateFlow, Navigation |
| Анимации          | Встроенный Compose Animation API                 |

### Декларативный принцип

```kotlin
// UI полностью определяется состоянием
@Composable
fun MoodScreen(
    uiState: MoodUiState,
    onScoreSelected: (Int) -> Unit
) {
    when (uiState) {
        is MoodUiState.Loading -> CircularProgressIndicator()
        is MoodUiState.Success -> MoodContent(
            entries = uiState.entries,
            onScoreSelected = onScoreSelected
        )
        is MoodUiState.Error -> ErrorMessage(uiState.message)
    }
}
```
## Альтернативы

| Подход            | Причина отказа                                     |
|-------------------|----------------------------------------------------|
| XML + View        | Устаревший подход, много шаблонного кода           |
| XML + DataBinding | Сложно поддерживать, смешивает логику с разметкой  |
| Flutter           | Требует Dart — выходит за рамки Android-разработки |
| React Native      | JavaScript — не нативный Android                   |

## Последствия
### Положительные
- Нет XML layout файлов — вся UI логика в Kotlin 
- Простое переиспользование компонентов 
- Удобное preview в Android Studio
### Отрицательные
- Минимальная версия API 21 (Android 5.0)
- Некоторые старые библиотеки требуют обёрток
