# ADR-007: HTTP-клиент Android - Retrofit
 
Статус: ✅ Принято

---

## Контекст

Android-клиент должен взаимодействовать с REST API бэкенда.
Необходима библиотека для HTTP-запросов с поддержкой:
- Kotlin Coroutines (suspend-функции)
- Перехватчиков запросов (добавление JWT заголовка)
- Автоматической десериализации JSON

---

## Решение

Использовать **Retrofit 2** + **OkHttp** + **Gson**.

### Конфигурация

```kotlin
// Foundation слой
object NetworkModule {

    fun provideRetrofit(
        tokenProvider: TokenProvider
    ): Retrofit = Retrofit.Builder()
        .baseUrl(BuildConfig.API_BASE_URL)
        .client(buildOkHttpClient(tokenProvider))
        .addConverterFactory(GsonConverterFactory.create())
        .build()

    private fun buildOkHttpClient(
        tokenProvider: TokenProvider
    ): OkHttpClient = OkHttpClient.Builder()
        .addInterceptor(AuthInterceptor(tokenProvider))
        .addInterceptor(
            HttpLoggingInterceptor().apply {
                level = if (BuildConfig.DEBUG)
                    HttpLoggingInterceptor.Level.BODY
                else
                    HttpLoggingInterceptor.Level.NONE
            }
        )
        .connectTimeout(30, TimeUnit.SECONDS)
        .readTimeout(30, TimeUnit.SECONDS)
        .build()
}

// Декларативное описание API
interface MindFlowApiService {

    @POST("auth/login")
    suspend fun login(@Body request: LoginRequest): AuthResponse

    @GET("meditations")
    suspend fun getMeditations(): List<MeditationResponse>

    @GET("meditations/{id}")
    suspend fun getMeditationById(@Path("id") id: Long): MeditationResponse

    @POST("mood")
    suspend fun saveMoodEntry(@Body request: MoodEntryRequest): MoodEntryResponse

    @GET("mood")
    suspend fun getMoodHistory(
        @Query("days") days: Int = 30
    ): List<MoodEntryResponse>
}
```

## Обоснование

| Критерий        | Retrofit                                   |
|-----------------|--------------------------------------------|
| Стандарт        | Де-факто стандарт HTTP-клиента для Android |
| Coroutines      | Нативная поддержка suspend-функций         |
| Декларативность | API описывается аннотациями - минимум кода |
| Документация    | Огромное комьюнити и документация          |
| Перехватчики    | OkHttp Interceptor для JWT заголовка       |

## Альтернативы

| Библиотека      | Причина отказа                                |
|-----------------|-----------------------------------------------|
| Ktor Client     | Менее распространён для Android               |
| Volley          | Устаревший, нет нативной поддержки Coroutines |
| OkHttp напрямую | Много шаблонного кода без декларативного API  |

---

## Последствия
### Положительные
- Минимальный код для описания API 
- Автоматическая сериализация/десериализация JSON 
- Удобное логирование запросов в debug-режиме
### Отрицательные
- Зависимость от внешней библиотеки 
- При изменении API необходимо обновлять интерфейс
