# ADR-006: Локальное хранилище Android — Room

Статус: ✅ Принято

---

## Контекст

Приложение должно работать **офлайн**.
Необходимо локально:
- Кэшировать библиотеку медитаций
- Хранить записи настроения до синхронизации
- Хранить историю сессий
- Хранить JWT токены (через DataStore)

---

## Решение

Использовать **Room** — ORM-библиотеку Google поверх SQLite.

### Стратегия офлайн-работы

Пользователь создаёт запись 
            ↓
Room (sync_pending = true)
            ↓
UI обновляется немедленно
            ↓
      Есть интернет?
       ↙          ↘
     Да            Нет
     ↓              ↓
 POST API       WorkManager
   ↓           ждёт сети
 sync_pending
= false


### Пример DAO

```kotlin
@Dao
interface MoodEntryDao {

    @Query("""
        SELECT * FROM mood_entries
        WHERE user_id = :userId
        ORDER BY recorded_at DESC
        LIMIT :limit
    """)
    fun getByUser(
        userId: Long,
        limit: Int = 30
    ): Flow<List<MoodEntryDbEntity>>

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insert(entry: MoodEntryDbEntity)

    @Query("""
        UPDATE mood_entries
        SET sync_pending = 0
        WHERE id = :id
    """)
    suspend fun markSynced(id: Long)

    @Query("""
        SELECT * FROM mood_entries
        WHERE sync_pending = 1
    """)
    suspend fun getPendingSync(): List<MoodEntryDbEntity>
}
```

## Обоснование

| Критерий         | Room                                     |
|------------------|------------------------------------------|
| Официальность    | Google Jetpack — официальная библиотека  |
| Типобезопасность | Compile-time проверка SQL-запросов       |
| Coroutines       | Нативная поддержка Flow и suspend        |
| Миграции         | Встроенные автоматические миграции схемы |
| Тестируемость    | In-memory база для unit-тестов           |
---

## Альтернативы

| Библиотека        | Причина отказа                              |
|-------------------|---------------------------------------------|
| SQLite напрямую   | Много шаблонного кода, нет типобезопасности |
| Realm             | Проприетарная, сложная интеграция           |
| ObjectBox         | Менее распространена, меньше документации   |
| SharedPreferences | Только примитивы, не для сложных структур   |
---

## Последствия
### Положительные
- Офлайн-первый подход без дополнительных усилий 
- Flow из Room автоматически обновляет UI при изменении данных 
- Тестируется с in-memory базой
### Отрицательные
- Нужны маппинг-функции: Domain Entity ↔ Room Entity 
- Поле sync_pending требует фоновой синхронизации (WorkManager)
