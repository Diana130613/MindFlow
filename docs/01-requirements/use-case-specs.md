# СПЕЦИФИКАЦИИ ПРЕЦЕДЕНТОВ

## UC-04: Запуск сессии медитации

| Поле | Значение |
|------|----------|
| **ID** | UC-04 |
| **Название** | Запуск сессии медитации |
| **Актор** | Пользователь |
| **Цель** | Провести сеанс медитации и сохранить результат |
| **Предусловия** | Пользователь аутентифицирован, выбрана медитация |
| **Постусловия** | Сессия сохранена (локально и/или на сервере), прогресс обновлён |

### Основной поток
1. Пользователь открывает экран списка медитаций
2. Система отображает список медитаций (из кэша или с сервера)
3. Пользователь выбирает медитацию
4. Система отображает экран деталей медитации
5. Пользователь нажимает "Начать"
6. Система запускает таймер и отображает инструкции
7. Пользователь завершает медитацию
8. Система сохраняет сессию в Room DB
9. Система отправляет данные на сервер (если есть сеть)
10. Система обновляет статистику (стрик, общее время)
11. Система показывает экран завершения с поздравлением

### Альтернативные потоки
- **4а. Офлайн-режим:** Медитация загружается из Room DB
- **9а. Нет сети:** Сессия помечается как "ожидает синхронизации", отправляется при следующем подключении
- **7а. Пользователь прерывает:** Система спрашивает "Завершить досрочно?", сохраняет фактическую длительность

### Исключения
- **E1:** Медитация не найдена в кэше и нет сети → показать сообщение об ошибке
- **E2:** Ошибка сохранения → повторить попытку 3 раза, затем показать ошибку

## UC-05: Запись настроения в дневник

| Поле | Значение |
|------|----------|
| **ID** | UC-05 |
| **Название** | Запись настроения в дневник |
| **Актор** | Пользователь |
| **Цель** | Зафиксировать эмоциональное состояние с заметкой |
| **Предусловия** | Пользователь аутентифицирован |
| **Постусловия** | Запись о настроении сохранена, аналитика обновлена |

### Основной поток
1. Пользователь открывает вкладку "Дневник"
2. Система отображает экран записи настроения
3. Пользователь выбирает оценку (1-10 или emoji)
4. Пользователь вводит текстовую заметку (опционально)
5. Пользователь выбирает теги эмоций (тревога, радость, усталость и т.д.)
6. Пользователь нажимает "Сохранить"
7. Система валидирует данные
8. Система сохраняет запись в Room DB
9. Система синхронизирует с сервером
10. Система показывает подтверждение

### Бизнес-правила
- Не более 1 записи настроения в день (обновление последней)
- Оценка настроения: целое число от 1 до 10
- Максимальная длина заметки: 500 символов
